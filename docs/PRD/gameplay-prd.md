## 1. 概述

游戏时界面是玩家实际操控小鸟飞行、躲避障碍物的核心玩法界面。该界面在玩家点击开始按钮后进入，承载完整的游戏逻辑、物理引擎、计分系统与暂停控制。

## 2. 界面布局

游戏容器固定为 **320px × 480px**，居中于页面。

│                             │
│                             │

## 4. 主体角色（小鸟）

### 4.1 图片资源

- 图片路径：`assets/images/bird.png`
- 不使用换肤系统，固定使用该图片。

### 4.2 尺寸

- 渲染尺寸：**40px × 40px**（在 320×480 容器内视觉比例合适）。

### 4.3 初始位置

- 水平位置：`x = 80px`（距左边约 1/4 处），游戏过程中水平位置固定不变。
- 垂直位置：`y = 200px`（容器垂直中心偏上）。

### 4.4 物理引擎

#### 水平运动
- 水平速度固定为 **0**（小鸟不水平移动，障碍物向左移动模拟前进）。

#### 垂直运动（真实物理）

```
每帧更新：
  velocityY += GRAVITY        // 重力加速度累积
  y += velocityY              // 位置更新
```

| 参数 | 值 | 说明 |
|------|----|------|
| `GRAVITY` | `0.5` | 每帧重力加速度（px/frame²） |
| `JUMP_FORCE` | `-10` | 点击时赋予的初速度（负值=向上） |
| 最大下落速度 | `12` | 防止穿透，限制 velocityY 上限 |

- 速度平缓：重力值 0.5 确保上升和下落过程自然，不会过于突兀。

#### 旋转角度

旋转角度随垂直速度线性映射，模拟真实飞翔姿态：

```
angle = clamp(velocityY * 3, -30, 90)  // 单位：度
```

| velocityY | 旋转角度 | 视觉效果 |
|-----------|----------|----------|
| -10（起跳） | -30° | 机头朝上 |
| 0（顶点） | 0° | 水平 |
| +4 | +12° | 微微低头 |
| ≥+12（最大下落） | +90° | 垂直俯冲 |

- 旋转中心：图片中心点。
- Canvas 绘制时使用 `ctx.save()` / `ctx.translate()` / `ctx.rotate()` / `ctx.restore()`。

### 4.5 死亡判定

以下任一条件触发游戏结束：

1. 小鸟与任意障碍物发生碰撞（矩形碰撞检测，留 4px 容差）。
2. 小鸟 `y < 0`（飞出顶部）。
3. 小鸟 `y + BIRD_SIZE > CANVAS_HEIGHT`（落到底部）。

## 5. 障碍物系统（管道）

### 5.1 图片资源

| 图片 | 路径 | 用途 |
|------|------|------|
| 下段管道 | `assets/images/pipe_up.png` | 从底部向上延伸的管道 |
| 上段管道 | `assets/images/pipe_down.png` | 从顶部向下延伸的管道 |

### 5.2 尺寸

- 管道宽度：**60px**（固定）。
- 管道高度：动态计算（见各组合规则）。

### 5.3 三种组合形式

#### 组合 A：仅下段（pipe_up）

```
┌──────────────────────────┐
│                          │
│                          │
│                          │
│                          │
│          ████████        │  ← pipe_up，高度 H_bottom
└──────────────────────────┘
```

- `pipe_up` 贴紧容器底部，高度 `H_bottom` 在 **[120px, 200px]** 随机。
- 图片从底部向上绘制，`y = CANVAS_HEIGHT - H_bottom`。

#### 组合 B：仅上段（pipe_down）

```
┌──────────────────────────┐
│          ████████        │  ← pipe_down，高度 H_top
│                          │
│                          │
│                          │
│                          │
└──────────────────────────┘
```

- `pipe_down` 贴紧容器顶部，高度 `H_top` 在 **[120px, 200px]** 随机。
- 图片从顶部向下绘制，`y = 0`。

#### 组合 C：上下段同时出现（x 值一致）

```
┌──────────────────────────┐
│          ████████        │  ← pipe_down，高度 H_top
│                          │
│       ← GAP ≥ 80px →     │  ← 通道间隙
│                          │
│          ████████        │  ← pipe_up，高度 H_bottom
└──────────────────────────┘
```

- 上下段 x 坐标相同。
- 间隙规则：`GAP = CANVAS_HEIGHT - H_top - H_bottom`，要求 `GAP ≥ BIRD_SIZE * 2 = 80px`。
- 生成逻辑：
  ```
  H_top    = random(80, 180)
  H_bottom = random(80, 180)
  // 校验间隙
  while (CANVAS_HEIGHT - H_top - H_bottom < 80) {
    H_top    = random(80, 180)
    H_bottom = random(80, 180)
  }
  ```

### 5.4 三种组合的出现概率

| 组合 | 概率 |
|------|------|
| 仅下段 | 30% |
| 仅上段 | 30% |
| 上下同时 | 40% |

### 5.5 移动与生成

| 参数 | 值 |
|------|----|
| `PIPE_SPEED` | `3` px/frame（向左移动） |
| `PIPE_SPAWN_INTERVAL` | `1500` ms |
| 初始生成 x | `CANVAS_WIDTH + 10 = 330px` |
| 销毁条件 | `x + PIPE_WIDTH < 0` |

### 5.6 计分逻辑

- 当小鸟 x 坐标超过管道 x + PIPE_WIDTH 时，该管道组计为"已通过"。
- 每通过一组障碍物，得分 +1。
- 同一组障碍物只计分一次（使用 `scored` 标志位）。

## 6. 暂停 / 继续系统

### 6.1 按钮位置与尺寸

- 位置：游戏容器右上角，`top: 12px; right: 12px`。
- 尺寸：**44px × 44px**（满足最小触控目标）。
- z-index：`5`（始终在 Canvas 之上）。

### 6.2 按钮状态

| 游戏状态 | 显示图片 | 含义 |
|----------|----------|------|
| 游戏进行中 | `assets/images/button_pause.png` | 点击可暂停 |
| 游戏已暂停 | `assets/images/button_resume.png` | 点击可继续 |

### 6.3 暂停行为

点击暂停按钮后：

1. 取消当前 `requestAnimationFrame`，游戏循环停止。
2. 画面冻结在当前帧（Canvas 内容不清除）。
3. 按钮图片切换为 `button_resume.png`。
4. **屏蔽所有非按钮点击事件**：Canvas 的 `onClick` 不响应，键盘空格键不响应。
5. 显示半透明遮罩层（`rgba(0,0,0,0.3)`）覆盖 Canvas，视觉上表示暂停状态。

### 6.4 继续行为

点击继续按钮后：

1. 移除遮罩层。
2. 按钮图片切换回 `button_pause.png`。
3. 重新启动 `requestAnimationFrame`，游戏循环恢复。
4. 恢复 Canvas 点击和键盘响应。

## 8. 状态切换动画

### 8.1 开始游戏 → 游戏进行中

- 开始界面卡片以 GSAP `back.in` 缩小淡出（`scale: 0.9, opacity: 0, duration: 0.5`）。
- 动画完成后切换 `gameState` 为 `PLAYING`，启动游戏循环。

### 8.2 游戏进行中 → 游戏结束

- 游戏循环停止。
- 游戏结束面板以 GSAP `back.out` 弹入（`scale: 0.75→1, opacity: 0→1, duration: 0.6`）。

### 8.3 游戏结束 → 重新开始

- 游戏结束面板以 GSAP `back.in` 缩小淡出（`scale: 0.85, opacity: 0, duration: 0.45`）。
- 动画完成后重置所有游戏状态，重新启动游戏循环。

## 11. 技术实现要点

1. **Canvas 渲染**：所有游戏元素（小鸟、管道）通过 Canvas API 绘制，每帧 `clearRect` 后重绘。
2. **RAF 驱动**：使用 `requestAnimationFrame` 驱动游戏循环，目标 60fps。
3. **图片预加载**：进入游戏前预加载所有游戏资源，加载过程通过 Toast 提示用户。
4. **碰撞检测**：矩形 AABB 碰撞，留 4px 容差避免误判。
5. **暂停屏蔽**：通过 React state `isPaused` 控制事件响应，暂停时 Canvas onClick 和键盘监听均不触发跳跃。
6. **旋转绘制**：使用 `ctx.save()` → `ctx.translate(cx, cy)` → `ctx.rotate(angle)` → `ctx.drawImage(...)` → `ctx.restore()` 实现以中心点旋转。

# PRD 生成 Skill

## AI 执行角色
在生成 PRD 时，你应当扮演：
**飞跃鸟项目的高级产品经理**。你需要基于项目已有的愿景、功能、技术约束（遵循 `claude.md` 的全部内容），将模糊需求转化为可落地的产品需求文档。

## PRD 文档必须包含的章节
### 1. 用户角色（Persona）
至少定义 2~3 个目标用户角色，格式：
- **角色名称**：简述特征
- **核心目标**：在本功能中想达成什么
- **痛点/动机**：为什么需要这个功能

### 2. 功能概述
用 2-3 句话说明本功能要解决什么问题，与游戏整体的关系。

### 3. 交互与状态流程
- 触发条件
- 每一步操作对应的界面/动画反馈（所有状态切换必须使用 GSAP 动画，资源加载必须 Toast 提示）
- 如涉及换肤，说明皮肤文件路径规则（`assets/skins/`，命名 kebab-case）

### 4. 技术实现要点
- 明确指出会新增或修改 `js/` 下的哪些文件（如 `main.js`、`config.js`）
- 新增的常量、变量命名必须对齐项目规范（UPPER_SNAKE_CASE、camelCase）
- 只需列出本项目约束带来的特殊要求，无需重复 `claude.md` 已定死的通用规范

### 5. 验收标准
- [ ] 可逐条验证的功能/交互/性能条件

## 文档存放要求
生成后的 PRD 应保存在 `docs/PRD/` 目录，文件名使用功能英文缩写 + `-prd.md`（例：`docs/PRD/music-toggle-prd.md`）。

## 关键原则
- **一切技术建议、动画方案、文件路径、资源命名必须完全吻合 `claude.md` 中的现有约束**，不引入任何矛盾设计。
- 语言简洁、中文撰写、功能可测得。
